# Security Policy

This project is pre-1.0; security fixes are applied to the latest `main` only.

Please **do not** open a public issue for security problems. Report them
privately via GitHub's
[private vulnerability reporting](https://docs.github.com/en/code-security/security-advisories/guidance-on-reporting-and-writing-information-about-vulnerabilities/privately-reporting-a-security-vulnerability)
("Security" tab -> "Report a vulnerability"), including enough detail to
reproduce (affected route or command, inputs, impact).
